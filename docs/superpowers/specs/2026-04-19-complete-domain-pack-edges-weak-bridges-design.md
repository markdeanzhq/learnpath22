# complete-domain-pack-edges 弱连接最小桥接设计

## 1. 背景

当前 `machine_learning` 领域包已经处于 `v1.1.0` 基线，正式规则不再允许真正孤点，但图中仍存在“弱连接节点/弱连接子图”问题：

- 图规模：48 个节点，55 条 `REQUIRES` 边
- 当前弱连通分量数：2
- 当前最明显的弱连接子图：`ml_a10 -> ml_b08 -> ml_b07 -> ml_e04`
- 当前优化目标不再是“补齐 formal orphan”，而是让这些弱连接节点真正进入**当前运行时 target** 的祖先链，从而进入闭包、排序、解释与评测

本设计只针对当前运行时评测口径进行优化，默认关注以下真实 target 集：

- G1 `我想系统学习机器学习基础`：`ml_b10`、`ml_b01`、`ml_e05`、`ml_d08`、`ml_d04`
- G2 `理解梯度下降`：`ml_c05`
- G3 `我想搞懂逻辑回归为什么能做分类`：`ml_c09`、`ml_e08`、`ml_e07`、`ml_d01`、`ml_c11`

## 2. 目标

在不改变节点集、资源、模板、阶段规则的前提下，只通过修改 `backend/app/domain_packs/machine_learning/requires_edges.json` 完成以下目标：

1. 优先修复当前弱连接实践子图的主链接入问题
2. 让统计链与评估链进入真实 target 的反向闭包
3. 改善 G1/G3 的路径覆盖与答辩叙事
4. 本轮不以 G2 提升为目标，但必须保证 G2 不退化
5. 保持整图教学因果自洽，避免为了刷覆盖率引入语义别扭的硬前置

## 3. 范围与边界

### In Scope

- 修改 `backend/app/domain_packs/machine_learning/requires_edges.json`
- 对弱连接节点与弱连接子图新增少量高价值 `REQUIRES` 边
- 以当前运行时 target 为优化基线
- 用非退化指标和结构改进指标验证方案有效性

### Out of Scope

- 不修改 `nodes.json`
- 不修改 `resources.json`
- 不修改 `goal_templates.json`
- 不修改 `stages.json` / `stage_rules.json`
- 不修改前后端业务代码
- 不以“让所有 optional 节点都进入主叙事”为目标

### 关键约束

- 本轮采用“最小桥接”，不是全面补边
- 不设置 `G1/G3` 节点增长硬上限，但必须通过非退化与解释一致性约束防止路径失控膨胀
- 每条新增边都必须能用一句教学因果解释清楚
- 每条新增边都必须服务于真实 target 的祖先闭包，而不是只让图在视觉上更稠密

## 4. 选定方案：最小桥接

本轮固定采用“最小桥接”方案，不采用“实践稠密”或“热点挂接”方案。

### 4.1 最小桥接包

新增 5 条 `REQUIRES`：

1. `ml_b08 -> ml_e03`
   - 语义：数据预处理先于训练基线模型
   - 作用：把 `ml_a10/ml_b08` 接入通用建模主叙事

2. `ml_e04 -> ml_e07`
   - 语义：特征标准化先于分类案例实践
   - 作用：把 `ml_b07/ml_e04` 接入分类实践主叙事

3. `ml_a09 -> ml_d07`
   - 语义：方差概念进入偏差-方差分析
   - 作用：把统计基础链接入评估链

4. `ml_d07 -> ml_d08`
   - 语义：偏差与方差分析先于模型选择基础
   - 作用：把评估链变成 G1 真实 target 的祖先链

5. `ml_c06 -> ml_e05`
   - 语义：学习率是超参数调优基础的直接前置
   - 作用：把调参链更稳地接入 G1 真实 target

### 4.2 方案成立条件

该桥接包必须同时满足：

- 弱连通分量数从 2 降到 1
- 弱连接实践子图不再作为独立分量存在
- G1/G3 的 backbone coverage 与 main-path coverage 持平或提升
- G2 不退化
- 新增边进入真实 target 的反向闭包
- 解释链读起来仍符合教学因果

## 5. 为什么不选其他方案

### 5.1 不选“实践稠密”

“实践稠密”会继续加密 `e02/e04/e07/e08` 一带的关系，覆盖提升可能更明显，但更容易导致：

- G1/G3 路径显著变长
- 评测波动扩大
- 论文叙事从“必要桥接”变成“扩张式补边”

### 5.2 不选“热点挂接”

“热点挂接”会把弱节点直接挂到 `e03/e07/d08` 等热点目标附近，虽然见效快，但语义风险最高：

- 祖先关系容易失真
- 解释文案会显得牵强
- 答辩时容易被追问“为什么这个一定是前置”

因此本设计明确拒绝只为覆盖率服务的捷径式挂接。

## 6. 数据流影响

本轮虽然只改 `REQUIRES`，但会影响四条主链。

### 6.1 闭包链

`backend/app/planner/closure.py` 使用 `requires_rev_adj` 做反向闭包。只有真实 target 的祖先才会进入最终计划节点集。

### 6.2 排序链

进入闭包后的节点才会进入 `scoring.py` / `topology.py` 的排序与主路径覆盖统计，因此“只改善视觉连通性”没有意义。

### 6.3 解释链

`backend/app/services/explanation_service.py` 也是基于真实祖先集合生成依赖链解释。边方向本身就是答辩叙事的一部分。

### 6.4 评测链

`evaluate.py`、thesis validation 与后续论文表述会直接受到边变化影响，尤其是：

- M1 dependency satisfaction
- M5 stage closure
- backbone coverage
- main-path coverage
- 解释链的可读性与可辩护性

## 7. 验证口径

本轮验证不使用“节点数最多增加 N 个”的硬约束，而使用“结构改进 + 非退化”双重口径。

### 7.1 成功标准

1. 弱连通分量必须从 2 变为 1
2. 图完整性校验继续通过
3. G1/G3 的 backbone coverage 不下降
4. G1/G3 的 main-path coverage 不下降
5. G2 的关键指标不下降
6. M1 必须保持 1.0
7. M5 必须保持 1.0
8. 新增边必须进入至少一个运行时 target 的祖先闭包
9. 新增后的解释链必须仍然语义自洽

### 7.2 失败条件

任一条件发生即视为设计失败并应回退：

- 新增边只让图更密，但进不了真实祖先闭包
- G2 退化
- G1/G3 覆盖反降
- 图虽然连通，但解释链明显失真
- 新边让 optional 弱叶子被不自然地强行推入主叙事

## 8. PBT / 不变量设计

| 属性 | 定义 | 反例生成方式 |
|---|---|---|
| DAG 不变量 | 修改后图必须保持 DAG | 引入会形成环的候选边，验证完整性检查是否拒绝 |
| 闭包可达性 | 每条新增边的 `source` 必须能到达至少一个运行时 target | 对每条新增边的 `source` 做 descendants 检查，若不能通向真实 target 即为反例 |
| 解释一致性 | 解释链必须等于 `ordered_ids` 与真实祖先闭包的交集 | 重算 reverse closure，与 explanation 输出对比；若出现多讲/漏讲即为反例 |
| 弱连通分量单调性 | 弱连通分量数不能增加，目标是 2 -> 1 | 生成只改善局部但未桥接主图的边，验证分量数未改善即为反例 |
| G2 非退化 | G2 的关键指标不得低于基线 | 比较变更前后 G2 评测结果；若 coverage 或主链指标下降即为反例 |
| G1/G3 覆盖非退化 | G1/G3 的 backbone/main-path coverage 不能下降 | 对比评测快照，若任一下降即为反例 |
| 教学因果一致性 | 每条新增边都能被一句前置因果解释 | 人工审查边理由；若只能以“为了覆盖率”解释即为反例 |

## 9. 回滚策略

出现以下任一情况应立即放弃该桥接包：

1. `REQUIRES` 修改后无法通过完整性校验
2. 弱连通分量仍为 2
3. 新增边未进入真实 target 祖先链
4. G2 指标下降
5. G1/G3 覆盖下降
6. 解释链变得难以答辩说明

## 10. 实施前要保持固定的决策

为避免实施阶段再做临场决策，本设计固定以下结论：

- 优化基线：使用运行时真实 target，而不是模板 target
- 主方案：最小桥接
- G2 口径：本轮不以 G2 提升为目标，但必须不退化
- 实践子图桥接：采用 `e03 + e07` 双叙事接入
- 统计评估链接入：采用 `ml_a09 -> ml_d07 -> ml_d08`
- 不设 G1/G3 节点增长硬上限，改用非退化和解释一致性约束

## 11. 预期效果

在不改变图谱其他文件的前提下，本设计预期获得以下结果：

- 当前唯一明显孤立实践子图被桥接回主图
- G1/G3 更容易覆盖统计链、评估链与实践链
- 主路径覆盖与答辩叙事变得更连贯
- 图仍保持“小而可解释”的论文原型口径，而不是向稠密课程图膨胀
